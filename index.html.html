Project Idea:

My project idea involves my interest in sports, primarily Basketball, programming a simulation of a game/season/career. Simulating players strengths and weaknesses,
tendencies, and chemistry with teammates and potentially opponents. The project would have customizability, from being able to edit players, their attributes, tendencies,
number, description, etc. To also be able to create your own player, who would be drafted into the NBA, and be able to improve through training and experience. Furthermore,
I want it to also be open to expansion, adding features, and possibly other sporting leagues.

My motivation for the creation of such a project is due to my interests in basketball and the NBA. Creating a basketball simulation also piques my interest as it is a realistic
project idea in which I either already have the tools and knowledge to create such a program, or I can learn the required skills and knowledge or acquire the right programming
tools and software.

The project would start by creating the foundation of the entire project, simulating a singular basketball game. With the project initially being similar to that of a text based RPG.
The output would include the final score, along with the statistics of each individual player, which will be calculated with a mixture of RNG, and their attributes/tendencies.
This leads to the next step, which would be simulating an entire season, game by game. It would start with the focused team the user is viewing, then adding the simulation of background
games to further immerse the user. In between games, the user can have options to edit players in their team, changing attributes or tendencies of the players, or even changing the players
names and descriptions all together. After laying the groundwork for a working basketball game simulation, the next addition would be adding a sort of RPG career, in which the user can be
either a player entering the league, or the general manager of a franchise. Where they will have the freedom to edit the roster and develop players, but with realistic limitations. The
path where the user creates their own basketball player to enter the NBA will be customizable to what the user wants. And can control what type of player they become.
After the completion of the text based program, the next step is to add a simple user interface to the program, where there will be menus to guide the user to their intended place much
easier and more aesthetically pleasing than the text based system. The user interface will also include paths to quick play simulation games, along with fully customizable rosters, to
other game modes, such as both the player and general manager career. Included in the user interface will also have side panels, which will show stat leaders per game, and teams records in
the current season. These will change throughout the experience as games are played.

The tools and technology used would likely be in a programming language such as Python, as it’s not only one of the most common and accessible languages, but also one I am currently learning
in another subject, ‘Introduction to Programming’. I would use software such as PyCharm, or VSCode to begin the project with, but possibly move to something else further down the track.
These two softwares are free and open to use. Storing the data used in the simulations could be stored in an SQL database, using an open software such as SQLiteStudio.

The skills required would first be a moderately advanced Python knowledge and understanding, as the algorithms to create the software would be relatively complex after the foundations are set.
Also required would be some knowledge in database usage, and the SQL language, as they would also be used.
With the successful outcome of the project, the primary audience would be basketball fans. As it wouldn’t be a full game title, but rather a small simulation tool or widget. The project could
also be used as a base for further development if I ever desire to. To possibly practice other IT tools on, such as graphic creation and further database development, or practicing some
multiplayer aspects on it if I ever get into that section of IT.
